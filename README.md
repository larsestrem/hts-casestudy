# HTS Growth Strategy — case study site

A static, zero-JavaScript portfolio case study built for the leadership of Helicopter
Transport Services (HTS). Every page is `noindex` (unpublished demo).

## Layout
- `portfolio/` — editable source (structured; one folder per page; absolute `/portfolio/…`
  links). **Edit here.**
- `build/` — flat, single-folder deploy artifact (relative links) that Cloudflare Pages
  serves. **Generated by `flatten.py`; never hand-edit.**
- `flatten.py` — regenerates `build/` from `portfolio/`.
- `deploy.sh` — `flatten.py` + commit + push to `main` (Cloudflare auto-deploys).
- `CLAUDE.md` — project conventions (auto-loaded by Claude Code).
- `HANDOFF.md` — full context, the six growth levers, the canonical nav block, and
  copy-paste verify/deploy commands.

## Quick start
```bash
python3 flatten.py                 # rebuild ./build
cd build && python3 -m http.server 8001   # preview at http://localhost:8001/
./deploy.sh "what changed"         # rebuild, commit, push -> Cloudflare deploys
```

## Cloudflare Pages (one-time, git integration)
Connect this repo as a Pages project. Production branch: `main`. Framework preset: **None**.
Build command: **(leave empty)**. Build output directory: **`build`**. After that, every
push to `main` deploys automatically.
